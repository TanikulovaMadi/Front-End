IP адрестер боюнча маселелер


Суроо боюнча вариантым номер 1


Суроо:

1. Төмөндө эки хосттун IP адрестери жана subnet mask берилген. Бул эки хост бир subnet ичиндеби же жокпу? Чыгаруу жолун толук жазып, жообуңузду түшүндүрүңүз.
host 1 address: 10.10.10.20
host 2address: 10.10.10.185
subnet mask: 255.255.255.224


Жообу:
1)Host 1 address: 10.10.10.20
В двоичной системе: 00001010.00001010.00001010.00010100 -> 10.10.10.20
Subnet mask: 11111111.11111111.11111111.11100000 -> 255.255.255.224
Subnet address: 00001010.00001010.00001010.00000000 -> 10.10.10.0


2)Host 2 address: 10.10.10.185
В двоичной системе: 00001010.00001010.00001010.10111001 -> 10.10.10.185
Subnet mask: 11111111.11111111.11111111.11100000 -> 255.255.255.224
Subnet address: 00001010.00001010.00001010.10100000 -> 10.10.10.160


3)Результат subnet address у двух хостов не совпадают, поэтому эти два хоста не лежат в одной подсети.
Subnet address 1 host : 10.10.10.0
Subnet address 2 host : 10.10.10.160

4)Если подробнее обьснить, то у Subnet mask: 11111111.11111111.11111111.11100000 -> 255.255.255.224
Это 5 шт нулей в 11100000 = 224 -> 2^5 = 32 host
Это означает что в одной подсети должно быть 32 хоста. Если логически посмотреть то они не могут лежать в одной подсети:
Я размышляла таким образом:
1 подсеть : 10.10.10.0/32 - в этой подсети расположен host 1 address: 10.10.10.20 >
2 подсеть : 10.10.10.32/64
3 подсеть : 10.10.10.64/96
4 подсеть : 10.10.10.96/128
5 подсеть : 10.10.10.128/160
6 подсеть : 10.10.10.160/192 - в этой подсети расположен host 2address: 10.10.10.185>
.....
Жыйынтык: Бул эки хост бир subnet ичинде жатпайт.
